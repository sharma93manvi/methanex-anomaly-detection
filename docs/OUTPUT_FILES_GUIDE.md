# Output Files Guide

This document explains all the output files generated by the anomaly detection pipeline and how to interpret them.

## Overview

The pipeline generates **7 output files** in the `output/` directory:
- 2 CSV files (detailed results)
- 4 PNG files (visualizations)
- 1 Markdown file (summary report)

---

## 1. CSV Results Files

### `Asset_1_results.csv` and `Asset_2_results.csv`

**What they are:**
Complete processed data with all original sensor readings, engineered features, anomaly flags, and scores.

**Structure:**
- **Rows**: 8,761 (one per hour from 2025-01-01 to 2026-01-01)
- **Columns**: 200 columns organized into 5 categories

### Column Categories:

#### **Category 1: Timestamp (1 column)**
- `Timestamp`: Date and time of each record (hourly intervals)

#### **Category 2: Original Sensor Data (18 columns)**
Raw sensor readings from the original dataset:
- `Asset 1 HP - Disch Press Value`: High pressure discharge pressure
- `Asset 1 HP - Pressure & Ratio Value`: Pressure ratio
- `Asset 1 HP - Suct Press Value`: Suction pressure
- `Asset 1T - Speed Value`: Turbine speed
- `Asset 1T Steam Inlet flow Value`: Steam flow
- `Asset 2 - Disch Press Value`: Asset 2 discharge pressure
- `Asset 2 Pressure & Ratio Value`: Asset 2 pressure ratio
- `Asset 2T - Speed Value`: Asset 2 turbine speed
- `target Value`: Target pressure value
- `Ambient Temperature Value`: Ambient temperature
- ... and more

#### **Category 3: Engineered Features (~151 columns)**

**A. Residual (1 column)**
- `residual`: Difference between actual and target pressure
  - Formula: `HP Discharge Pressure - Target Value`
  - Key indicator of process excursions

**B. Rate of Change (8 columns)**
- `{sensor}_roc`: First derivative (change per hour)
  - Examples: `Asset 1 HP - Disch Press Value_roc`, `Asset 1T - Speed Value_roc`
  - Identifies sudden changes or trends

**C. Rolling Statistics (~56 columns)**
- `{sensor}_rolling_{stat}_{window}`: Moving statistics
  - Statistics: `mean`, `std`, `min`, `max`
  - Windows: `6h` (6 hours), `24h` (24 hours)
  - Examples:
    - `Asset 1 HP - Disch Press Value_rolling_mean_6h`
    - `residual_rolling_std_24h`
  - Captures recent and daily patterns

**D. Time Features (4 columns)**
- `hour`: Hour of day (0-23)
- `day_of_week`: Day of week (0-6)
- `month`: Month (1-12)
- `day_of_year`: Day of year (1-365)

**E. Normalized Features (~82 columns)**
- `{feature}_normalized`: Z-score normalized versions
  - All numeric features have normalized versions
  - Used for ML models (ensures fair weighting)

#### **Category 4: Anomaly Flags (13 columns)**

**Statistical Method Flags:**
- `anomaly_residual`: Flagged by residual-based detection
- `anomaly_zscore_{sensor}`: Flagged by rolling z-score (4 sensors)
- `anomaly_envelope_{sensor}`: Flagged by moving average envelope (4 sensors)
- `anomaly_percentile_{sensor}`: Flagged by percentile method (4 sensors)

**Combined Flags:**
- `anomaly_statistical`: **True** if ANY statistical method flagged (requires 3+ hour persistence)
  - Aligns with notification escalation threshold for consistency
- `anomaly_ml`: **True** if ML ensemble flagged (top 5% of scores)
  - **Immediate detection** - no persistence requirement
  - ML methods catch anomalies as soon as they occur
- `anomaly_combined`: **True** if EITHER statistical OR ML flagged (OR logic)
  - **Used by notification system** - triggers immediate warnings
  - Combines both approaches for comprehensive detection

**Values**: `True` = anomaly detected, `False` = normal

#### **Category 5: Anomaly Scores (7 columns)**

**Statistical Scores:**
- `anomaly_score_statistical`: Combined score from all statistical methods (0-1)
  - Higher = more anomalous
  - Threshold: varies, but typically >0.5 for sustained anomalies

**ML Scores:**
- `anomaly_score_isolation_forest`: Isolation Forest score (0-1)
- `anomaly_score_autoencoder`: Autoencoder reconstruction error (0-1)
- `anomaly_score_lstm`: LSTM prediction error (0-1)
- `anomaly_score_ml`: **Ensemble ML score** (weighted average)
  - Weights: 70% Isolation Forest, 15% Autoencoder, 15% LSTM
  - Threshold: ~0.39 (95th percentile = top 5%)

**Combined Score:**
- `anomaly_score_combined`: Average of statistical and ML scores (0-1)

**Score Interpretation:**
- **0.0-0.2**: Normal operation
- **0.2-0.4**: Slightly unusual
- **0.4-0.6**: Moderate anomaly
- **0.6-0.8**: Strong anomaly
- **0.8-1.0**: Severe anomaly

### How to Use the CSV Files:

1. **Filter Anomalies:**
   ```python
   import pandas as pd
   df = pd.read_csv('Asset_1_results.csv')
   anomalies = df[df['anomaly_combined'] == True]
   print(f"Found {len(anomalies)} anomalies")
   ```

2. **Analyze Specific Time Periods:**
   ```python
   df['Timestamp'] = pd.to_datetime(df['Timestamp'])
   april_anomalies = df[(df['Timestamp'].dt.month == 4) & (df['anomaly_combined'] == True)]
   ```

3. **Check Score Distributions:**
   ```python
   print(df['anomaly_score_ml'].describe())
   print(f"Threshold: {df['anomaly_score_ml'].quantile(0.95)}")
   ```

4. **Investigate Specific Sensors:**
   ```python
   # Find which sensors were anomalous at specific times
   anomaly_times = df[df['anomaly_combined'] == True]['Timestamp']
   # Check sensor values at those times
   ```

## 2. Visualization Files

### `Asset_1_time_series.png` and `Asset_2_time_series.png`

**What they show:**
Time series plots of key sensors with anomalies highlighted.

**Content:**
- **3 subplots** showing:
  1. HP Discharge Pressure over time
  2. Pressure & Ratio Value over time
  3. Speed Value over time
- **Red dots/markers**: Anomaly points flagged by the system
- **Colored lines**: Normal sensor readings (color varies by matplotlib's default palette, typically pink/light colors)
- **X-axis**: Time (Jan 2025 - Jan 2026)
- **Y-axis**: Sensor values

**How to interpret:**
- **Red markers** indicate when anomalies were detected
- **Colored lines** show the sensor values over time (normal operation)
- **Patterns**: Look for clusters of red markers (persistent issues)
- **Context**: See sensor behavior before/during/after anomalies
- **Validation**: Verify that red markers correspond to unusual sensor behavior

### `Asset_1_scores.png` and `Asset_2_scores.png`

**What they show:**
Anomaly scores over time from different detection methods.

**Content:**
- **Multiple lines** showing:
  - Statistical anomaly score (blue)
  - ML ensemble score (orange/red)
  - Threshold line (dashed, ~0.39)
- **X-axis**: Time
- **Y-axis**: Anomaly score (0-1)

**How to interpret:**
- **Peaks above threshold**: Anomalies detected
- **Statistical vs ML**: Compare which method catches what
- **Score magnitude**: Higher peaks = more severe anomalies
- **Temporal patterns**: See if anomalies cluster in time

## Tips for Interpretation

1. **Anomaly Flags vs Scores:**
   - Flags (`True`/`False`) = binary decision
   - Scores (0-1) = confidence/severity

2. **Statistical vs ML:**
   - Statistical: Conservative, high precision (0-5 anomalies)
   - ML: Broader detection (439 anomalies, 5.01%)

3. **Early Detection:**
   - Top sensors provide 18-20 hour lead time
   - Use these for proactive monitoring

4. **Score Thresholds:**
   - ML threshold: ~0.39 (top 5%)
   - Adjust if you want more/fewer detections

5. **Time Patterns:**
   - Anomalies distributed across year
   - Some periods have more (April-September 2025)
